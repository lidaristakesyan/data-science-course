Types of Data

Structured Data

  • Organized in tables with rows and columns (databases, spreadsheets)
  • Examples: Sales records, customer databases, sensor readings
  • Easy to analyze with SQL and traditional tools

Unstructured Data

  • No predefined format (text, images, videos, audio)
  • Examples: Social media posts, emails, medical images
  • Requires preprocessing and specialized techniques (NLP, Computer Vision)

Semi-Structured Data

  • Has some organizational properties (JSON, XML, logs)
  • Examples: Web logs, configuration files, API responses

Common Data Quality Issues

Missing Values

Data points that are absent or incomplete. Can occur due to errors, non-responses, or system issues.

Outliers

Extreme values that differ significantly from other observations. Can be errors or genuine anomalies.

Duplicates

Repeated records that can skew analysis and inflate metrics.

Inconsistencies

Conflicting data across sources or within the same dataset.

Invalid Data

Values that don't make sense (e.g., negative age, future dates).

Formatting Issues

Different date formats, units, or encodings across datasets.

Exploratory Data Analysis (EDA)

What is EDA?

The process of analyzing data sets to summarize their main characteristics, often using visual methods. It's about understanding your data before modeling.

Key EDA Techniques

  • Summary Statistics: Mean, median, mode, standard deviation, quartiles
  • Distributions: Histograms, box plots, density plots
  • Relationships: Scatter plots, correlation matrices, pair plots
  • Patterns: Time series plots, seasonal decomposition
  • Anomalies: Outlier detection, unusual patterns

Event-Driven Architecture

An event represents something meaningful that happened at a specific point in time, capturing what, when, who, and context.

Event Components

What Happened?

Event type (click, purchase, view)

When?

Timestamp (precise time of event)

Who/What?

User ID, session ID, device info

Context

Page URL, product ID, metadata

Key Design Decisions

  • Granularity: How detailed should event tracking be?
  • Timing: Synchronous (immediate) vs Asynchronous (delayed)
  • Schema: Structured vs flexible event properties

Best Practices: Working with Event Data

  • Always understand data provenance
    Know where your data comes from, how it was collected, and what transformations were applied
  • Question data quality early
    Check for missing events, sampling bias, timing issues, and collection artifacts
  • Document assumptions
    Record what you know about the data collection process and its limitations
  • Consider privacy implications
    Ensure compliance with regulations and ethical use of user data
  • Validate across multiple sources
    Cross-reference data from different systems to verify accuracy
  • Plan for data evolution
    Build pipelines that can handle schema changes and new event types

How HDFS Works

  • NameNode: Manages metadata and coordinates file operations
  • DataNodes: Store actual data blocks (typically 128 MB each)
  • Replication: Each block replicated 3x by default (fault tolerance)
  • Write-once, read-many: Optimized for large sequential reads

Spark Core Concepts

RDD (Resilient Distributed Dataset)

Immutable distributed collection of objects that can be processed in parallel. Low-level API.

DataFrame

Distributed collection of data organized into named columns (like a database table). High-level API with optimizations.

Dataset

Type-safe version of DataFrame (available in Scala/Java, not Python). Best of both RDD and DataFrame.
For Python: Use DataFrames! They're the most optimized and easiest to work with in PySpark.

Why Spark is Fast: In-Memory Computing

‚ùå MapReduce (Disk-Based)

Iteration 1:
Read from HDFS ‚Üí Process ‚Üí Write to HDFS

Iteration 2:
Read from HDFS ‚Üí Process ‚Üí Write to HDFS

Iteration 3:
Read from HDFS ‚Üí Process ‚Üí Write to HDFS

Result: Very slow due to disk I/O

‚úÖ Spark (In-Memory)

Load once:
Read from HDFS ‚Üí Cache in RAM

Iterations 1-N:
Process from RAM ‚Üí Results in RAM

Final step:
Write results to HDFS

Result: 10-100x faster!
